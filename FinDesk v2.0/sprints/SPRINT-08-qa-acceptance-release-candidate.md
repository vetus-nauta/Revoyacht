# Sprint 08 — QA, Acceptance, and MVP Release Candidate

## Goal

Prove FinDesk v2.0 Clean Core MVP is complete.

## Depends on

Sprint 07 final report.

## Director rule

Use a new Director. Start from project docs and Sprint 07 handoff only.

## Agents

- QA/Audit Agent
- Financial Logic Agent as reviewer
- Backend Core Agent as reviewer
- UX Layout Agent as reviewer
- Frontend Performance Agent as reviewer
- Security/Privacy Agent if real data is present

## Scope

- all fixtures;
- arithmetic tests;
- parser tests;
- import tests;
- responsive tests;
- keyboard tests;
- iOS-like feel review;
- Definition of Done check;
- final Director report.

## Forbidden

- no new features;
- no redesign;
- no formula changes without Director approval;
- no accepting partial MVP as done.

## Exit criteria

- all critical tests pass;
- Definition of Done has no blocker;
- release candidate report exists;
- postponed work is separated from MVP;
- next phase list is documented separately.

## Final handoff

Include:

- release candidate status;
- complete test report;
- known limitations;
- future phase list;
- files changed;
- final Director recommendation.
