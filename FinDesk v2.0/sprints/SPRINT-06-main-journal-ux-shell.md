# Sprint 06 — Main Journal UX Shell

## Goal

Build the main journal screen around the approved core.

## Depends on

Sprint 05 final report.

## Director rule

Use a new Director. Start from project docs and Sprint 05 handoff only.

## Agents

- iOS-Native UX Layout Agent
- Frontend Performance and Interaction Agent
- QA/Audit Agent
- Financial Logic Engine Agent as reviewer only

## Scope

- current month notes-style feed;
- input bar;
- Cash/Card switch;
- visible invalid rows;
- Other expenses visual state;
- entry detail panel or drawer;
- mobile note feed;
- mobile structured horizontal view;
- full workspace layout for desktop and iPad 11+.

## Forbidden

- no new finance rules;
- no page/body scroll;
- no dashboard-first screen;
- no narrow phone layout on desktop;
- no hidden input when keyboard opens.

## Exit criteria

- feed visible during entry;
- input reachable with keyboard open;
- desktop uses available width;
- iPad 11+ uses full workspace layout;
- iPad mini uses mobile behavior;
- phone vertical scrolls notes;
- phone horizontal view shows structured rows;
- QA confirms layout contract.

## Handoff to Sprint 07

Include responsive status, keyboard risks, UX blockers, unfinished device checks, and files changed.
