# Sprint 07 — Integration Hardening and Attachments

## Goal

Connect MVP pieces and harden the product without expanding scope.

## Depends on

Sprint 06 final report.

## Director rule

Use a new Director. Start from project docs and Sprint 06 handoff only.

## Agents

- Data and Backend Core Agent
- Frontend Performance and Interaction Agent
- QA/Audit Agent
- Security/Privacy Agent if real data is used

## Scope

- auth wiring if safe;
- workspace member roles;
- attachment upload and metadata;
- admin shell minimum;
- audit log verification;
- category reassignment;
- saved category rules after approval;
- integration tests.

## Forbidden

- no new large features;
- no forecast;
- no full archive import;
- no dashboard expansion;
- no old finance logic.

## Exit criteria

- workspace permissions work;
- attachments connect to entries;
- category reassignment works;
- audit log records meaningful changes;
- integration tests pass;
- no legacy logic leaks in.

## Handoff to Sprint 08

Include integration status, unresolved security issues, attachment status, role status, tests, and files changed.
